## Завдання 0: Базовий клас Model

Створити `classes/Model.php` — абстрактний клас з інтерфейсом для всіх моделей:

```php
abstract class Model {
    protected static string $table;        // назва таблиці
    protected static array $fillable;      // дозволені поля для mass-assignment

    public static function all(): array;           // SELECT * FROM <table>
    public static function find(int $id): ?static; // SELECT * WHERE id = ?
    public static function where(string $col, $val): array; // SELECT * WHERE col = ?
    public function save(): bool;                  // INSERT або UPDATE (залежно від $this->id)
    public function delete(): bool;                // DELETE WHERE id = ?
    public function fill(array $data): void;      // $this->title = $data['title']
}
```

**Вимоги:**
- Використовує `Database::getInstance()` **тільки тут** — контролери не знають про PDO
- Prepared statements у кожному запиті
- Повертає об'єкти, а не асоціативні масиви (`Recipe` об'єкт, не `['title' => ...]`)
- `save()` сам вирішує INSERT/UPDATE за наявністю `$this->id`

- Формат: **JSON Lines** (`data/comments.jsonl`) — Sub3 формат

### 1.2 Завантаження зображень

- JPEG, PNG, GIF, WebP, до 5 МБ → `data/uploads/`
- Галерея всіх завантажених зображень
- **MVC:** `UploadController` НЕ робить `move_uploaded_file` прямо — викликає `Image::store($file)` з моделі

### 1.3 Робота з каталогами

- Форма створення: **Логін** + **Пароль** → створити `data/users/{login}/` з підпапками `video`, `music`, `photo` + по 2-3 файли в кожній (порожні `.txt`)

- Окрема сторінка видалення: **Логін** + **Пароль** → перевірити що логін збігається з тим що в `data/users/{login}/.auth` (або зберігати хеш пароля в окремому файлі) → видалити папку рекурсивно з усім вмістом

## Завдання 2: База даних з моделями

### 2.1 Модель Recipe + CRUD

Таблиця `recipes`:

| Поле | Тип | Опис |
|------|-----|------|
| id | INT (PK, AUTO_INCREMENT) | |
| title | VARCHAR(200) | Назва |
| category | VARCHAR(100) | сніданок/обід/вечеря/десерт/закуска |
| cooking_time | INT | хвилини |
| servings | INT | порції |
| ingredients | TEXT | Список інгредієнтів |
| user_id | INT (FK → users.id) | Автор рецепту |
| created_at | DATETIME | |

**Модель `models/Recipe.php`:**

```php
class Recipe extends Model {
    protected static string $table = 'recipes';
    protected static array $fillable = ['title','category','cooking_time','servings','ingredients','user_id'];

    // Зв'язок: автор рецепту
    public function author(): ?User {
        return User::find($this->user_id);
    }

    // Зв'язок: коментарі (One-to-Many)
    public function comments(): array {
        return Comment::where('recipe_id', $this->id);
    }

    // Бізнес-метод
    public function isQuick(): bool {
        return $this->cooking_time <= 30;
    }
}
```

**Контролер `controllers/RecipeController.php`:**

```php
public function index(Request $req) {
    $recipes = Recipe::all();             // НЕ $db->query()
    return View::render('recipes/list', compact('recipes'));
}

public function store(Request $req) {
    $recipe = new Recipe();
    $recipe->fill($req->post());           // title, category, ...
    $recipe->user_id = $_SESSION['user_id'];
    $recipe->save();                       // INSERT
    Router::redirect('/recipes');
}
```

**⛔ Заборонено у контролері:** `$db = Database::getInstance()`, `$pdo->prepare(...)`, будь-який `SELECT/INSERT/UPDATE/DELETE`.

### 2.2 Модель User + авторизація

Таблиця `users` (10 полів — Sub3 base):

| Поле | Тип |
|------|-----|
| id | INT (PK) |
| login | VARCHAR(50) UNIQUE |
| password | VARCHAR(255) |
| email | VARCHAR(100) |
| first_name | VARCHAR(50) |
| last_name | VARCHAR(50) |
| phone | VARCHAR(20) |
| city | VARCHAR(50) |
| gender | VARCHAR(10) |
| about | TEXT |
| created_at | DATETIME |

**Правила (Sub3):**
- Унікальний логін
- Пароль: **≥5 символів + мінімум одна цифра**
- Email формат
- `password_hash(PASSWORD_DEFAULT)` + `password_verify`

**Модель `models/User.php`:**

```php
class User extends Model {
    protected static string $table = 'users';
    protected static array $fillable = ['login','email','first_name','last_name','phone','city','gender','about'];

    public static function findByLogin(string $login): ?self {
        $results = self::where('login', $login);
        return $results[0] ?? null;
    }

    public function setPassword(string $plain): void {
        $this->password = password_hash($plain, PASSWORD_DEFAULT);
    }

    public function checkPassword(string $plain): bool {
        return password_verify($plain, $this->password);
    }

    // Зв'язок: рецепти користувача
    public function recipes(): array {
        return Recipe::where('user_id', $this->id);
    }
}
```

**Сторінки:** реєстрація, вхід, профіль, редагування, видалення, вихід — через `AuthController` + `User` модель.

### 2.3 Модель Comment (НОВЕ в v31)

Таблиця `comments` — природне розширення кулінарного блогу:

| Поле | Тип | Опис |
|------|-----|------|
| id | INT (PK) | |
| recipe_id | INT (FK → recipes.id) | До якого рецепту |
| user_id | INT (FK → users.id) | Хто написав |
| parent_id | INT (FK → comments.id, NULL) | Для тредів (null = корінь) |
| text | TEXT | |
| created_at | DATETIME | |

**Модель `models/Comment.php`:**

```php
class Comment extends Model {
    protected static string $table = 'comments';
    protected static array $fillable = ['recipe_id','user_id','parent_id','text'];

    public function author(): ?User    { return User::find($this->user_id); }
    public function recipe(): ?Recipe  { return Recipe::find($this->recipe_id); }
    public function replies(): array   { return Comment::where('parent_id', $this->id); }

    public static function tree(int $recipeId): array {
        // Корневі коментарі (parent_id IS NULL) для рецепта
        $roots = self::whereNull('parent_id', $recipeId);
        // Для кожного кореня — рекурсивно додати replies
        foreach ($roots as $root) {
            $root->loadReplies();  // заповнює $root->replies масивом об'єктів Comment
        }
        return $roots;
    }
}
```

**Деталі треду:**
- **Макс. глибина — 2 рівні** (корінь → відповідь). Відповісти на відповідь не можна (форма `reply` доступна тільки коли `$comment->parent_id === null`)
- **UI на сторінці рецепту `/recipes/{id}`:**
  1. Під рецептом — форма "Залишити коментар" (доступна тільки залогіненим)
  2. Список корневих коментарів з автором, датою, текстом
  3. Під кожним коренем — вкладений список відповідей (візуальний відступ `margin-left: 2em`)
  4. Під кожним корінним коментарем — кнопка "Відповісти" → показує форму з hidden `parent_id`
- **Видалення:** автор коментаря або автор рецепту може видалити коментар. Видалення кореня каскадно видаляє всі відповіді (`ON DELETE CASCADE` у FK `parent_id` — див. schema.sql нижче)

### 2.4 Сесії та Cookie (з ЛР4)

**Сесія — колір фону:**
- Сторінка `/settings` з формою (`<select>` 5 кольорів теми) → POST → `$_SESSION['bg_color'] = $req->post('bg_color')` → redirect back
- Всі layouts читають `$_SESSION['bg_color']` і застосовують до `<body style="background-color: <?= htmlspecialchars($bg) ?>">`
- Дефолтний колір якщо сесія порожня: персиковий `#FFF3E0`

**Cookie — привітання:**
- При першому візиті: показати форму "Як до вас звертатись?" → POST → `setcookie('greeting', $name, time()+30*24*3600)` (30 днів) → redirect
- При наступних візитах: якщо `$_COOKIE['greeting']` існує → показати `"Вітаємо, {ім'я}!"` у шапці
- Кнопка "Змінити ім'я" → `setcookie('greeting', '', time()-3600)` → знову форма

**Кольори (з v30, Sub3 палітра):**

| Колір | Hex | Тема |
|-------|-----|------|
| Персиковий | `#FFF3E0` | Смачний десерт |
| Зелений | `#F1F8E9` | Свіжі трави та овочі |
| Рожевий | `#FCE4EC` | Ягідні нотки |
| Жовтий | `#FFFDE7` | Прянощі та куркума |
| Лавандовий | `#EDE7F6` | Прованський стиль |

**MVC:** сесії та cookies — системні примітиви, залишаються в контролерах (не в моделі). Контролер читає/пише `$_SESSION`/`$_COOKIE` → передає значення у View як звичайну змінну.

## ⛔ Правила MVC separation

### ❌ ЗАБОРОНЕНО

| Де | Що | Приклад порушення |
|----|-----|-------------------|
| Controller | SQL, `Database::getInstance()`, `PDO` | `$db->query("SELECT * FROM recipes")` |
| View | SQL, бізнес-логіка, запити до БД | `<?php foreach (Recipe::all() as ...) ?>` — ОК, але `<?php $pdo->query(...) ?>` — ні |
| Model | `echo`, HTML, `header()`, доступ до `$_POST`/`$_GET` | `echo "<h1>...</h1>"` всередині `Recipe::all()` |

### ✅ ДОЗВОЛЕНО

| Де | Що |
|----|-----|
| Controller | `Recipe::all()`, `$recipe->save()`, `Router::redirect()`, передача даних у View |
| View | `<?php echo htmlspecialchars($recipe->title) ?>`, цикли по переданим даним |
| Model | `Database::getInstance()`, prepared statements, бізнес-методи, зв'язки |

## Database schema (`database/schema.sql`)

```sql
CREATE TABLE users (
    id           INT PRIMARY KEY AUTO_INCREMENT,
    login        VARCHAR(50) UNIQUE NOT NULL,
    password     VARCHAR(255) NOT NULL,
    email        VARCHAR(100) NOT NULL,
    first_name   VARCHAR(50)  NOT NULL,
    last_name    VARCHAR(50)  NOT NULL,
    phone        VARCHAR(20)  NOT NULL,
    city         VARCHAR(50)  NOT NULL,
    gender       VARCHAR(10)  NOT NULL,
    about        TEXT         NULL,
    created_at   DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE recipes (
    id            INT PRIMARY KEY AUTO_INCREMENT,
    title         VARCHAR(200) NOT NULL,
    category      VARCHAR(100) NOT NULL,
    cooking_time  INT          NOT NULL,
    servings      INT          NOT NULL,
    ingredients   TEXT         NOT NULL,
    user_id       INT          NOT NULL,
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_recipes_user (user_id),
    INDEX idx_recipes_category (category)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE comments (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    recipe_id   INT  NOT NULL,
    user_id     INT  NOT NULL,
    parent_id   INT  NULL,              -- NULL = корінь треду
    text        TEXT NOT NULL,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (recipe_id) REFERENCES recipes(id)  ON DELETE CASCADE,
    FOREIGN KEY (user_id)   REFERENCES users(id)    ON DELETE CASCADE,
    FOREIGN KEY (parent_id) REFERENCES comments(id) ON DELETE CASCADE,
    INDEX idx_comments_recipe (recipe_id),
    INDEX idx_comments_parent (parent_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## Entry point `index.php`

```php
<?php
// 1. Bootstrap
require_once __DIR__ . '/classes/Application.php';
Application::boot(__DIR__);              // autoload, session_start, error_reporting

// 2. Autoloader для моделей (classes/ + controllers/ + models/)
spl_autoload_register(function (string $cls): void {
    foreach (['classes', 'controllers', 'models'] as $dir) {
        $path = __DIR__ . "/$dir/$cls.php";
        if (file_exists($path)) { require_once $path; return; }
    }
});

// 3. Connection check (fail fast)
try { Database::getInstance(); }
catch (PDOException $e) {
    http_response_code(500);
    exit('Database connection failed. Перевірте config/database.php');
}

// 4. Router (код вище)
// ...
```

## Валідація даних

Валідація — у окремому класі `classes/Validator.php` (або статичні методи моделі). **Контролер** викликає валідацію **перед** `save()`. Якщо помилки → render форми з `$errors`.

### Recipe

| Поле | Правило |
|------|---------|
| `title` | required, 3-200 символів |
| `category` | required, ∈ {сніданок, обід, вечеря, десерт, закуска} |
| `cooking_time` | required, integer, 1-1440 (≤24 год) |
| `servings` | required, integer, 1-50 |
| `ingredients` | required, 10-5000 символів |

### User (реєстрація)

| Поле | Правило |
|------|---------|
| `login` | required, 3-50 символів, `[a-zA-Z0-9_]+`, унікальний (`User::findByLogin()` === null) |
| `password` | required, **≥5 символів + мінімум одна цифра** (`preg_match('/\d/', $pwd) && strlen($pwd) >= 5`) |
| `email` | required, `filter_var(..., FILTER_VALIDATE_EMAIL)` |
| `first_name`, `last_name`, `city` | required, 1-50 символів |
| `phone` | required, `preg_match('/^\+?\d{10,15}$/', $phone)` |
| `gender` | required, ∈ {male, female, other} |
| `about` | optional, ≤1000 символів |

### Comment

| Поле | Правило |
|------|---------|
| `text` | required, 1-2000 символів, `strip_tags` перед збереженням |
| `parent_id` | optional, якщо задано → `Comment::find($parent_id)?->parent_id === null` (не можна відповідати на відповідь) |

**Shape:**
```php
$errors = Validator::validate($req->post(), [
    'title' => ['required', 'min:3', 'max:200'],
    // ...
]);
if (!empty($errors)) return View::render('recipes/form', compact('errors', 'recipe'));
```

## Authorization (Policy)

Політика доступу — у класі `classes/Policy.php` (або методах моделі). Контролер перевіряє **перед** дією.

| Дія | Правило |
|-----|---------|
| `POST /recipes` | авторизований (session має `user_id`) |
| `GET /recipes/{id}/edit`, `POST /recipes/{id}`, `POST /recipes/{id}/delete` | `$_SESSION['user_id'] === $recipe->user_id` |
| `POST /recipes/{id}/comments` | авторизований |
| `POST /comments/{id}/delete` | `$_SESSION['user_id'] === $comment->user_id` **АБО** `$_SESSION['user_id'] === $comment->recipe()->user_id` (автор рецепту модерує) |
| `GET /profile`, редагування профілю, видалення акаунту | авторизований, тільки власний профіль |

**Порушення → `http_response_code(403)` + `exit('Forbidden')`** (або redirect на `/login` якщо не залогінений).

```php
// Приклад у RecipeController
public function update(Request $req, int $id): void {
    $recipe = Recipe::find($id) ?? abort(404);
    Policy::recipeOwner($recipe);           // 403 якщо не власник
    Policy::csrf($req);                     // 419 якщо токен невалідний
    $errors = Validator::validate($req->post(), Recipe::rules());
    // ...
}
```

## CSRF захист

Кожна POST-форма **ОБОВ'ЯЗКОВО** містить CSRF токен. Реалізація в `classes/Csrf.php`:

```php
class Csrf {
    public static function token(): string {
        if (empty($_SESSION['csrf'])) {
            $_SESSION['csrf'] = bin2hex(random_bytes(32));
        }
        return $_SESSION['csrf'];
    }
    public static function verify(string $token): bool {
        return !empty($_SESSION['csrf']) && hash_equals($_SESSION['csrf'], $token);
    }
}
```

**У View (форма):**
```php
<form method="post" action="/recipes">
    <input type="hidden" name="_csrf" value="<?= htmlspecialchars(Csrf::token()) ?>">
    <!-- ... -->
</form>
```

**У контролері (перед state-mutating action):**
```php
if (!Csrf::verify($req->post('_csrf') ?? '')) {
    http_response_code(419);
    exit('CSRF token mismatch');
}
```

**Форми, що вимагають CSRF:** register, login, logout, recipe create/update/delete, comment create/delete, profile update, settings, guestbook, upload, folder create/remove.

## Error handling

**PDO:** у `classes/Database.php` (singleton) встановити режим `ERRMODE_EXCEPTION`:
```php
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
```

**У моделях:** кожен метод з PDO обгорнути `try/catch (PDOException $e)`:
```php
public function save(): bool {
    try {
        $stmt = Database::getInstance()->prepare($sql);
        return $stmt->execute($params);
    } catch (PDOException $e) {
        error_log("Recipe::save() failed: " . $e->getMessage());
        return false;
    }
}
```

**У контролерах:** перевіряти результат `save()`/`delete()`. Якщо `false` → показати форму з `$errors[] = 'Помилка збереження, спробуйте ще раз'`.

**Глобальний handler** у `index.php`:
```php
set_exception_handler(function (Throwable $e): void {
    error_log($e);
    http_response_code(500);
    require __DIR__ . '/views/errors/500.php';
});
```

**Views помилок:** `views/errors/404.php`, `views/errors/403.php`, `views/errors/419.php`, `views/errors/500.php`.

## Чек-лист приймання v31

**MVC структура:**
- [ ] Існує `classes/Model.php` з інтерфейсом (`all`, `find`, `where`, `save`, `delete`, `fill`)
- [ ] Існує папка `models/` з **3 моделями**: `Recipe.php`, `User.php`, `Comment.php`
- [ ] Жодного SQL у `controllers/` (`grep -rE "SELECT|INSERT|UPDATE|DELETE|Database::" controllers/` → 0 збігів)
- [ ] Жодного `echo`/HTML у `models/` (`grep -rE "echo|<\?=|<[a-z]+>" models/` → 0 поза коментарями/docblocks)
- [ ] Жодних запитів до БД у `views/` (`grep -rE "Database|PDO|->query|->prepare" views/` → 0)
- [ ] Усі 3 сутності мають FK-зв'язки: `Recipe::author()`, `Recipe::comments()`, `User::recipes()`, `Comment::replies()`, `Comment::author()`

**База даних:**
- [ ] `database/schema.sql` існує з 3 таблицями + FK + ON DELETE CASCADE
- [ ] `database/seed.php` заповнює 5 users + 10 recipes + 15 comments
- [ ] Database::getInstance() має `ERRMODE_EXCEPTION`
- [ ] Кожен метод моделі з PDO обгорнутий у `try/catch (PDOException $e)`

**Безпека:**
- [ ] Хешування паролів через `password_hash(PASSWORD_DEFAULT)` (не MD5/SHA1)
- [ ] Пароль при реєстрації: мінімум 5 символів + містить цифру (regex `/\d/`)
- [ ] **CSRF токен** у всіх state-mutating формах (register, login, logout, recipe CRUD, comments, profile, settings, guestbook, upload, folder)
- [ ] `Csrf::verify()` у кожному відповідному контролері, 419 при невдачі
- [ ] `htmlspecialchars()` у всіх views при виводі user-input даних (XSS захист)
- [ ] Prepared statements у 100% SQL запитів (`grep -rE "->query\(" models/` не має конкатенації зі змінними)

**Валідація:**
- [ ] Клас `Validator` або метод `::rules()` у моделях
- [ ] Recipe: title 3-200, category з enum, cooking_time 1-1440, servings 1-50, ingredients 10-5000
- [ ] User: login 3-50 `[a-zA-Z0-9_]+` унікальний, password ≥5 + цифра, email формат, phone regex
- [ ] Comment: text 1-2000, `strip_tags`, `parent_id` не глибше 1 рівня
- [ ] Помилки показуються у формі з `$errors` + `$old` (передзаповнення полів)

**Authorization:**
- [ ] `Policy::recipeOwner()` блокує редагування/видалення чужих рецептів (403)
- [ ] Коментар видаляє автор коментаря АБО автор рецепта
- [ ] Неавторизовані: 302 на `/login` для protected actions
- [ ] Профіль — тільки власний

**Функціонал:**
- [ ] CRUD сторінки для рецептів працюють (list, create, edit, delete) через Model
- [ ] Коментарі з тредами: можна відповідати на корневі коментарі (parent_id), глибина максимум 2
- [ ] Гостьова у форматі JSONL (`data/comments.jsonl`)
- [ ] Завантаження зображень: валідація MIME + розміру, галерея
- [ ] Каталоги: створення + видалення з паролем
- [ ] Session (колір фону) + Cookie (привітання 30 днів) з ЛР4 працюють

**Error handling:**
- [ ] `views/errors/{404,403,419,500}.php` існують
- [ ] Global `set_exception_handler` у `index.php`
- [ ] Error log через `error_log()` на PDOException

## Міграція з v30 → v31

Якщо студент уже зробив v30 і хоче перейти на v31 (щоб мати каркас курсової):

1. **Скопіювати папку:** `cp -r variants/v30 variants/v31` (або створити нову з нуля).
2. **Додати `classes/Model.php`** — базовий клас (див. § Завдання 0).
3. **Створити папку `models/`:**
   - Перенести з кожного контролера SQL у відповідну модель:
     - `IndexController::list()` з `SELECT * FROM recipes` → `Recipe::all()`
     - `CatalogController::edit()` з `UPDATE recipes SET ...` → `$recipe->save()`
     - `AuthController::register()` з `INSERT INTO users ...` → `$user->save()`
   - Створити `models/Recipe.php`, `models/User.php`.
4. **Додати `models/Comment.php`** + таблицю `comments` (нова сутність у v31).
5. **Оновити `database/schema.sql`** — додати FK + індекси + таблицю `comments`.
6. **Додати `classes/Csrf.php`** + вставити `<input type="hidden" name="_csrf">` у кожну форму.
7. **Додати `classes/Validator.php`** — замінити `if (empty($_POST['title']))` перевірки на централізовану валідацію.
8. **Додати `classes/Policy.php`** — замінити прямі перевірки `$_SESSION['user_id']` у контролерах на `Policy::recipeOwner()`.
9. **Додати `views/errors/` + global exception handler** у `index.php`.
10. **Прогнати grep-перевірки** з Чек-листа — має бути 0 збігів SQL у controllers/views та 0 echo у models.
11. **Прогнати `database/seed.php`** → перевірити що 5 users + 10 recipes + 15 comments з'явились.

**Оцінка часу:** ~6-10 годин роботи для студента, який уже має v30.
